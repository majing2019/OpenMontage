# Documentary Montage Pipeline — 检索式蒙太奇流水线

> Last updated: 2026-07-12 | Documentary Montage v1.0

## 概况

Documentary Montage 是 OpenMontage 的**检索优先、编辑驱动的蒙太奇流水线**——不生成 AI 画面，而是从互联网公开素材库中**搜索真实视频片段**，通过语义匹配和叙事编排，打造散文电影式的短片。

| 特性 | 说明 |
|------|------|
| 名称 | `documentary-montage` |
| 目标 | 给定一个主题/情感/问题，自主检索素材 → 编排 → 调色 → 配乐 → 合成 |
| 核心差异化 | CLIP 语义检索、16 个免费素材源、五情绪基调系统、哲理结尾卡、L-Cut 声效焊接 |
| 最终产出 | `.mp4` 视频（Remotion CinematicRenderer 合成） |
| 时长范围 | 30-180 秒 |
| 音乐 | **强制**——除非用户明确要求无声 |
| 旁白 | 可选，默认无声 |
| 稳定性 | beta |

## 适合什么视频

### ✅ 推荐场景

| 类型 | 典型时长 | 范例主题 |
|------|---------|---------|
| **散文电影 / 音画诗** | 60-180s | "回家的感觉是什么？" |
| **主题蒙太奇** | 30-90s | "雨中的城市"、"地球上所有的脚印" |
| **情绪拼贴画** | 30-60s | 纯情感驱动的影像序列——怀旧、孤独、敬畏 |
| **时代档案混剪** | 60-180s | "20 世纪如何想象未来"——用 Prelinger 档案片 |
| **社会观察短片** | 60-120s | "每个凌晨三点还在工作的人"——目录式积累 |
| **儿童童话/睡前故事** | 60-180s | Pixabay AI 生成奇幻动画 + 可选旁白 |
| **品牌文化短片** | 60-120s | "我们制造的每件东西都来自同一个地方" |

### ❌ 不适合

- 结构化讲解/科普视频 → 用 `animated-explainer`
- AI 生成动画/动态图形 → 用 `animation`
- 预告片/宣传片（以用户素材为主）→ 用 `cinematic`
- 访谈/人物口播 → 用 `talking-head`
- 含人物对白的叙事短片 → 用 `story-factory`

## 五情绪基调系统

在 idea 阶段锁定**一个**基调，所有下游决策（剪辑节奏、调色、素材选择）都以此校准。

| 基调 | 平均镜头时长 | 转场风格 | 调色倾向 | 适合主题 |
|------|-------------|---------|---------|---------|
| **elegiac**（挽歌式） | 4.0s | 硬切为主，偶用慢溶解 | 暖色、褪色感、柔焦边缘 | 记忆、失去、故乡、时间流逝 |
| **urgent**（紧迫式） | 1.2s | 纯硬切，节奏驱动 | 高对比、冷色、锐利 | 危机、城市、新闻、当下事件 |
| **reverent**（敬畏式） | 3.5s | 硬切，对称构图 | 自然色、暗部压低 | 自然奇观、仪式、宏观尺度 |
| **wry**（讽刺式） | 2.0s | 硬切，靠并置制造笑点 | 微褪色、档案质感 | 消费文化、政治讽刺、怀旧乐观 |
| **dreamlike**（梦幻式） | 3.0s | 慢溶解、重复母题 | 柔光、过曝、饱和度降低 | 童年回忆、悲伤、潜意识 |

### 基调 × 时长速查表

| 基调 | 30s 镜头数 | 60s 镜头数 | 90s 镜头数 | 180s 镜头数 |
|------|-----------|-----------|-----------|------------|
| elegiac | ~8 | ~15 | ~23 | ~45 |
| reverent | ~9 | ~17 | ~26 | ~51 |
| dreamlike | ~10 | ~20 | ~30 | ~60 |
| wry | ~15 | ~30 | ~45 | ~90 |
| urgent | ~25 | ~50 | ~75 | ~150 |

## 四种叙事结构

| 结构 | 描述 | 推荐时长 | 范例 |
|------|------|---------|------|
| **list**（目录式） | 无弧线，纯积累——"每个曾经____的人" | 30-90s | "凌晨四点的城市"——25 个画面并列 |
| **before/after**（转折式） | 前半段建立 → 后半段翻转 | 60-120s | "人类如何想象未来 vs 实际发生了什么" |
| **three-act**（三幕式） | 建立(30%) → 转折(40%) → 释放(30%) | 90-180s | Adam Curtis 风格深度散文 |
| **single-image expansion**（单意向展开） | 一个画面反复从多角度呈现 | 30-60s | "门"——各种门：打开、关上、等待、离开 |

## 16 个免费素材源

### 现代高清（默认主干）

| 来源 | 内容特点 | 分辨率 |
|------|---------|--------|
| **Pexels** | 现代高清，人物、城市、自然、商业 | HD/4K |
| **Pixabay Video** | 大型社区库，含 AI 生成奇幻动画 | HD/4K |
| **Coverr** | 精选影视级 B-roll，自然、都市 | HD/4K |
| **Mixkit** | Envato 精选，自然、科技、商业 | HD/4K |
| **Videvo** | 90K+ 免费素材，自然、航空、延时 | HD/4K |
| **Dareful** | 精品 4K 自然——山脉、森林、瀑布、延时 | 4K |
| **Unsplash** | 编辑级静帧图片（动态不足时的补充） | 高分辨率 |

### 档案/历史（复古质感）

| 来源 | 内容特点 | 年代范围 |
|------|---------|---------|
| **Archive.org**（Prelinger） | 家庭录像、教育片、工业宣传片 | 1940s-1980s |
| **NARA**（美国国家档案馆） | 二战、冷战、阿波罗、民权、总统 | 1900s-1980s |
| **Library of Congress** | 早期电影、新闻片、文化记录 | 1928 年前公有领域 |
| **Pond5 PD** | 一战/二战、早期电影、历史演讲 | 1900s-1940s |

### 专题/科学

| 来源 | 内容特点 |
|------|---------|
| **NASA** | 地球轨道影像、天文、飞行、宏观尺度 |
| **ESA** | 欧洲航天——哈勃/韦伯影像、地球观测 |
| **JAXA** | 日本航天——隼鸟号、ISS 希望号实验舱 |
| **NOAA** | 深海 ROV 影像、珊瑚礁、海洋生物、飓风 |

### 素材源选择提示

- **`era_mix: "modern"`** → 偏向 Pexels/Pixabay/Coverr/Mixkit
- **`era_mix: "vintage"`** → 偏向 Archive.org/NARA/LOC/Pond5 PD（至少 60% 来自档案源）
- **`era_mix: "any"`** → 按镜头语义混合，场景导演逐 slot 决定
- **儿童/童话内容** → 锁定 Pixabay Video（AI 奇幻动画库），不混用真实素材

## 如何启动一条 Documentary Montage

### 第一步：告诉 Agent 你的主题

最简单的启动方式——只需要一个主题：

```
"帮我做一个关于'城市凌晨四点'的蒙太奇短片"
"做一个像 Adam Curtis 风格的视频，关于'技术乐观主义的失败'"
"用档案片做一个关于'回家'的散文短片，60秒，挽歌风格"
```

### 第二步：Agent 会做 preflight 并提问

Agent 会运行工具发现，展示你的能力菜单，然后引导你完成 idea 阶段：

1. **提炼主题问题**：Agent 从你的描述中提炼出一句话的 thematic question
2. **选择情绪基调**：5 种基调中选一个
3. **确定时长和结构**：30s? 90s? list? three-act?
4. **制定音乐计划**：用你 `music_library/` 里的曲目？还是生成？还是你提供？
5. **提出结尾卡**：一句 3-9 词的主题句，叠加在最终画面或黑场上

### 第三步：审核并批准 idea

Agent 产出 `brief` 后，会呈现给你审核：
- 主题问题是否准确
- 基调是否对味
- 时长和结构是否合理
- 音乐来源是否可行
- 结尾卡文字是否点题

### 第四步：Agent 自主执行

批准后，Agent 依次自动完成：

1. **场景规划** → 分解为具体画面 slot，写 CLIP 搜索描述
2. **素材检索** → 从 16 个源搜索 + 语义排序 + 逐 slot 选片
3. **编辑编排** → 定入出点 + 转场 + 音乐同步 + 结尾卡时机
4. **合成导出** → Remotion 渲染 `.mp4`

每个阶段 Agent 会做自查，关键决策点会请求你确认。

## 快速路径 vs 标准路径

素材检索阶段有两条路径可选：

### 快速路径（`direct_clip_search`）— 适合 < 30 slots

```
用户搜索 → 直接调用各素材源 API → 下载 2-3 个候选/slot → 肉眼审缩略图 → 选片
```

- **优点**：快速迭代，分幕输出，支持跨幕复用已下载素材
- **适用**：快速试验、分幕制作、< 30 个镜头的作品
- **并行**：搜索下载的同时可以并行处理 TTS 旁白、字幕、音乐

### 标准路径（`corpus_builder` + `clip_search`）— 适合 50+ slots

```
批量搜索 → 建 CLIP 嵌入索引 → 逐 slot 用画面描述做语义排序 → 自动选片 → 去重
```

- **优点**：全自动，CLIP 语义匹配精准，适合大规模
- **适用**：50+ slots 的大制作，需要自动化排重
- **候选池规则**：语料库大小应 ≥ 8x slot 数量

### 如何选择

Agent 会在 preflight 后根据你的 slot 数量和需求推荐路径。你也可以明确指定：

```
"用快速路径，我想分幕看结果"
"用标准路径，全自动跑"
```

## 关键设计决策

### 哲理结尾卡（End Tag）—— 强制

每条 documentary-montage 视频都以一段 3-9 词的哲理文字结尾（Remotion EndTag 组件渲染）。这不是制作人员名单——是一句点睛的主题句。

**默认模式是 `overlay`**（ProRes 4444 带 alpha 通道叠加在最后几秒画面之上），让结尾自然融入。备选模式是 `concat`（黑场 + 文字，拼接在正片之后）。

只有用户明确说"不要结尾卡"时才会跳过——Agent 会记录明确的 opt-out。

### 音乐 —— 强制

纪录片蒙太奇音乐不是可选项。没有音乐的素材拼贴看起来像被遗弃的半成品。唯一的例外是用户明确说"我要无声"。

Agent 会在 idea 阶段就解决音乐问题，不会拖到素材阶段制造意外。

### 旁白 —— 可选

与音乐和结尾卡不同，旁白是可选的。纯画面 + 音乐完全能撑起来。如果需要旁白，会在 idea 阶段指定 TTS 提供方和声音。

## 编辑器技法速览

| 技法 | 说明 | 使用频率 |
|------|------|---------|
| **硬切** | 默认转场，大部分剪辑都是硬切 | 90%+ |
| **溶解** | 0.5-1.0s，用于情感关联镜头或时间流逝 | 每片 1-3 次 |
| **淡入/淡出** | 片头片尾，0.5-1.0s | 每片 2 次 |
| **音乐留白** | 在情绪中心位置静音 ~2s | 每片 1 次（≥60s 时） |
| **L-Cut** | 前镜头环境音（雨声、脚步）延续到后镜头 0.5-1.5s | 最难过渡的 3-4 处 |
| **Hero 镜头延长** | 标记为 hero 的 slot 镜头时长翻倍 | 每片 2-3 处 |

**禁忌**：划像、推拉、缩放模糊、RGB 分离、漏光、故障效果——这些是社交媒体编辑语言，会破坏纪录片质感。

## 完整使用示例

### 示例 1：最简单的启动

```
用户: "帮我做一个关于'雨中的城市'的短片，90秒"

Agent:
  → 运行 preflight，展示能力菜单
  → 提炼 thematic question: "雨在城市中揭示了什么？"
  → 推荐基调: elegiac
  → 推荐结构: list，15 个 slot，每个 ~6s
  → 检查 music_library/，推荐曲目
  → 提议结尾卡: "这座城市守着自己的夜。"
  → 等待用户批准

用户: "基调用 elegiac，音乐用你推荐的，结尾卡很好，开始吧"

Agent:
  → 场景规划: 15 个 slot——雨滴打沥青、门廊下撑伞、霓虹倒映水洼...
  → 素材检索: 从 Pexels + Archive.org 搜索，CLIP 排序，逐 slot 选片
  → 编辑编排: 硬切为主，hero 镜头 6-7s，54s 处音乐留白 2s
  → 合成导出: Remotion 渲染，统一暖色 LUT，结尾卡叠加
  → 输出: projects/rain-in-the-city/renders/final.mp4
```

### 示例 2：复古档案风格

```
用户: "做一个关于'50年代美国郊区生活'的蒙太奇，用档案片，
      讽刺风格，60秒，wry 基调"

Agent:
  → era_mix = "vintage"，锁定 Archive.org / NARA / LOC / Pond5 PD
  → 写 period-appropriate 搜索词: "commuter", "housewife", "suburb"
    （不用 "influencer", "wfh", "coworking"）
  → slot 设计侧重并置的荒诞感: 广告笑容 × 真实家庭 × 核弹演习
  → 以 ironic juxtaposition 驱动剪辑
```

### 示例 3：儿童童话

```
用户: "做一个关于'毛毛虫变蝴蝶'的短片给孩子看，60秒，梦幻感"

Agent:
  → 检测到 children's content → 切换素材策略
  → 锁定 Pixabay Video → AI 生成奇幻动画
  → 重写所有 query 加 fantasy 前缀:
    "fairy tale caterpillar magical forest glowing"
    "enchanted chrysalis magical tree"
    "fantasy butterfly glowing magical wings"
  → 不混用真实素材——保持视觉一致性
```

## 常见陷阱

| 陷阱 | 表现 | 正确做法 |
|------|------|---------|
| **主题漂移** | brief 说的是"回家"，slot list 全是飞机 | 写完 slot 后回读 brief |
| **情绪词当画面描述** | "温暖的欢迎时刻"——CLIP 无法理解 | 写名词和形容词："女人站在门廊，手靠近门把" |
| **单 query per slot** | 第一个 query 搜出垃圾，就没有备选 | 每个 slot 写 2-3 个不同角度的 query |
| **跳过音乐计划** | idea 阶段没定音乐，到素材阶段才发现没得用 | idea 阶段就检查 music_library/ 和可用 API |
| **按分数盲选** | 只看 CLIP 分数最高的，不管基调是否匹配 | 霓虹闪烁的拉斯维加斯街景 ≠ 挽歌基调，分数再高也不用 |
| **忘记 exclude_ids** | 同一个精彩镜头赢了两三个 slot | 维护已选 clip_id 黑名单 |
| **过渡词汇太花** | 溶解、划像、缩放——社交媒体的视觉语言 | 最多 4 种转场类型，90% 硬切 |
| **结尾画面弱** | 最后一个镜头无足轻重 | 最后一个镜头被记忆的权重极高——不行就换 |
| **静帧结尾** | 最后一帧 freeze——看起来像技术失误 | 用 fade_to_black 或让画面自然结束 |
